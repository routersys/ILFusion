# ILFusion

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](#)
[![.NET](https://img.shields.io/badge/.NET-10.0-purple.svg)](#)
[![Release](https://img.shields.io/github/v/release/routersys/ILFusion.svg)](https://github.com/routersys/ILFusion/releases)

---

[ILRepack](https://github.com/gluck/il-repack) を手軽に実行するための、**対話式 TUI でアセンブリを選択して結合できる .NET 10 製 CLI ツール**です。
対象ディレクトリを指定するだけで DLL/EXE を自動検出し、主アセンブリの単一選択・結合対象の複数選択という 2 段階の対話型 UI で直感的に操作できます。

`ilrepack /out:merged.dll primary.dll secondary.dll ...` を手入力する代わりに、パスを一度渡すだけで自動リストアップから結合実行まで完結します。

## 概要

ILFusion は、ILRepack のコマンドライン引数を手動で構築する手間を解消するための対話型ラッパーツールです。

指定したディレクトリ（またはアセンブリのパス）から DLL / EXE を自動検出し、対話型 TUI で **主アセンブリ → 結合対象** の順に選択するだけで、ILRepack を正しい引数で実行します。

依存アセンブリの解決に失敗する場合は、ライブラリ検索パス（`/lib:`）を追加で指定することもできます。ソースディレクトリは常に自動追加されるため、多くの場合は追加設定なしで動作します。

---

## 動作要件

| 項目 | 要件 |
|---|---|
| OS | Windows / Linux / macOS (x64 / ARM64) |
| ILRepack | PATH 上に `ILRepack`（または `ILRepack.exe`）が存在すること |

ILFusion 本体は NativeAOT でコンパイルされた単一実行ファイルであり、**.NET ランタイムのインストールは不要**です。

---

## インストール方法

1. [Releases](https://github.com/routersys/ILFusion/releases/latest) ページから対象プラットフォームの実行ファイルをダウンロードしてください。

| ファイル名 | 対象プラットフォーム |
|---|---|
| `ilfusion-win-x64.exe` | Windows (x64) |
| `ilfusion-win-arm64.exe` | Windows (ARM64) |
| `ilfusion-linux-x64` | Linux (x64) |
| `ilfusion-linux-arm64` | Linux (ARM64) |
| `ilfusion-osx-x64` | macOS (Intel) |
| `ilfusion-osx-arm64` | macOS (Apple Silicon) |

2. 実行ファイルを PATH の通ったディレクトリに配置します。
3. Linux / macOS では実行権限を付与してください。

```sh
chmod +x ilfusion-linux-x64
```

4. 別途 [ILRepack](https://github.com/gluck/il-repack/releases) を入手し、同様に PATH の通った場所に配置してください。ILFusion と同じディレクトリに配置した場合も自動的に検出されます。

## 基本的な使い方

### ディレクトリを指定して起動

```sh
ilfusion C:\MyApp\bin\Release\net10.0
```

### アセンブリのパスを指定して起動

ファイルパスを指定した場合、そのファイルが主アセンブリとして自動設定されます。

```sh
ilfusion C:\MyApp\bin\Release\net10.0\MyApp.dll
```

### 引数なしで起動

引数を省略した場合は、対話形式でパスの入力を求められます。

```sh
ilfusion
```

```
ディレクトリまたはアセンブリのパスを入力してください:
> C:\MyApp\bin\Release\net10.0
```

## 対話型 UI の操作方法

### 1. 主アセンブリの選択

ディレクトリを指定した場合、最初に主アセンブリを 1 つ選択する画面が表示されます。主アセンブリは ILRepack の第 1 引数として渡され、エントリーポイントや参照情報の基準となります。

ファイルパスを指定して起動した場合は、この画面をスキップして次の手順に進みます。

```
【主アセンブリの選択】

[↑↓] 移動  [Enter] 決定  [Esc] キャンセル

 ▶  MyApp.dll                     (1.2 MB)
    HelperLibrary.dll             (345 KB)
    Newtonsoft.Json.dll           (678 KB)
    SomeUtility.dll               (120 KB)
```

| キー | 動作 |
|---|---|
| `↑` / `↓` | カーソル移動 |
| `Enter` | 選択して確定 |
| `Esc` | キャンセル（終了） |

### 2. 結合対象の選択

主アセンブリが確定すると、残りのアセンブリから結合対象を複数選択する画面に移ります。選択済みのアセンブリは緑色で表示されます。

```
【結合するアセンブリの選択】

[Space] 選択/解除  [↑↓] 移動  [Ctrl+A] 全選択/解除  [Enter] 確定  [Esc] キャンセル

   [○]  HelperLibrary.dll         (345 KB)
 ▶ [●]  Newtonsoft.Json.dll       (678 KB)
   [○]  SomeUtility.dll           (120 KB)
```

| キー | 動作 |
|---|---|
| `↑` / `↓` | カーソル移動 |
| `Space` | 選択 / 解除 |
| `Ctrl+A` | 全選択 / 全解除 |
| `Enter` | 選択中のアセンブリで確定（1 つ以上の選択が必要） |
| `Esc` | キャンセル（終了） |

### 3. 出力パスとオプション設定

結合対象が確定すると、順番に以下の設定を対話形式で入力します。いずれも未入力のままで Enter を押すとデフォルト値が適用されます。

```
出力ファイルのパスを入力してください:
  (未入力の場合: C:\MyApp\bin\Release\net10.0\MyApp.merged.dll)
> 

オプション設定 (y/N):
  内部化する (/internalize) [y/N]: 
  デバッグ情報を除去する (/ndebug) [y/N]: 
  ユニオンマージ (/union) [y/N]: 

追加のライブラリ検索パスを入力してください:
  依存アセンブリが別ディレクトリにある場合に指定します (空行で完了)
> 
```

すべての設定が完了すると ILRepack が実行されます。

---

## オプション一覧

| オプション | ILRepack 引数 | 説明 |
|---|---|---|
| 内部化 | `/internalize` | 結合後、主アセンブリ以外の型を `internal` に変更する |
| デバッグ情報の除去 | `/ndebug` | デバッグシンボルを除去し、ファイルサイズを削減する |
| ユニオンマージ | `/union` | 同名の型が複数のアセンブリに存在する場合にマージする |
| ライブラリ検索パス | `/lib:<path>` | 依存アセンブリの解決に使用する追加の検索ディレクトリ |

ソースディレクトリ（入力アセンブリの存在するディレクトリ）はライブラリ検索パスに**常に自動追加**されます。

---

## 終了コード

| コード | 意味 |
|---|---|
| `0` | 正常終了（結合成功） |
| `1` | 指定したパスが見つからない |
| `2` | 対象ディレクトリにアセンブリが存在しない |
| `3` | 選択されたアセンブリが不足している |
| `4` | ILRepack の実行に失敗した |

## 注意事項

- **ILRepack は別途インストールが必要**: ILFusion は ILRepack のラッパーであり、ILRepack 本体を同梱していません。PATH 上に `ILRepack`（Windows では `ILRepack.exe`）が存在しない場合、実行に失敗します。ILFusion と同じディレクトリに配置した場合は自動的に検出されます。
- **依存アセンブリの解決エラー**: ILRepack が `Failed to resolve assembly` エラーを出力した場合は、オプション設定の「追加のライブラリ検索パス」に該当アセンブリが含まれるディレクトリを指定してください。
- **macOS の実行制限**: macOS ではダウンロードした実行ファイルに Gatekeeper の制限がかかる場合があります。`xattr -d com.apple.quarantine ./ilfusion-osx-arm64` で制限を解除できます。
- **Linux の実行権限**: ダウンロード後に `chmod +x` で実行権限を付与してください。
- **ILRepack の出力はそのまま表示される**: 実行後の標準出力・標準エラーは ILRepack のものがそのまま表示されます。詳細なエラー内容は ILRepack のドキュメントを参照してください。
